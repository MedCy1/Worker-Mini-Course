Web Workers API

Parallélisme dans le navigateur, sans bloquer l'UI

Le problème

JavaScript est mono-thread

Le navigateur exécute JS, le rendu DOM et les événements utilisateur sur un seul fil d'exécution. Une tâche lourde gèle tout.

Sans Worker
Calcul lourd (CPU)
⏸ Rendu UI bloqué
⏸ Clics ignorés
⏸ Animations figées
Avec Worker
Main thread : UI fluide
Worker thread : calcul
Clics réactifs
Animations à 60 fps
La solution

Qu'est-ce qu'un Web Worker ?

Un script JavaScript qui s'exécute dans un thread séparé, communiquant avec la page principale via un système de messages.

🌐

Main Thread

Page / UI
postMessage() →
← onmessage
Messages copiés (structuredClone)
⚙️

Worker Thread

Calcul isolé
Taxonomie

3 types de Workers

👤

Dedicated Worker

Lié à un seul script. Le cas d'usage le plus courant. Terminé avec la page.

🤝

Shared Worker

Partagé entre plusieurs onglets/iframes de même origine. Communication via un port.

🛡️

Service Worker

Proxy réseau persistent. Gère le cache, le offline et les push notifications.

Interface

L'API en pratique

new Worker('worker.js')
Crée et démarre un worker depuis un fichier JS.
worker.postMessage(data)
Envoie un message (données copiées via structuredClone).
worker.onmessage = fn
Reçoit la réponse du worker dans le thread principal.
self.onmessage = fn
Côté worker : écoute les messages entrants.
self.postMessage(result)
Renvoie le résultat au thread principal.
worker.terminate()
Stoppe immédiatement le worker.
Cas d'usage

Quand utiliser Web Workers ?

🔢

Calculs intensifs

Chiffrement/déchiffrement, compression, traitement d'images pixel par pixel, simulations physiques.

📊

Traitement de données massives

Parsing de gros fichiers JSON/CSV, tri et filtrage de larges datasets en temps réel.

🎮

Jeux & applications temps-réel

Physique, IA, pathfinding déportés dans un worker pour garder 60 fps sur le rendu.

À savoir

Possibilités & Limites

✅ Disponible dans le Worker

  • fetch / XMLHttpRequest
  • setTimeout / setInterval
  • WebSockets
  • IndexedDB
  • Crypto API
  • Import de modules (type module)

❌ Non disponible dans le Worker

  • Accès direct au DOM
  • Objet window
  • document / alert / confirm
  • localStorage / sessionStorage
  • Certaines API média
